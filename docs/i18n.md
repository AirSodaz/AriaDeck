# AriaDeck i18n

**Stack:** Fluent (`.ftl`) via `fluent-bundle` · crate `ariadeck-i18n` (no GPUI) · catalogs `crates/ariadeck-i18n/i18n/{en,zh-CN}/*.ftl`

## Locales

| Settings `language` | Catalog |
| --- | --- |
| `system` | OS UI lang (`sys-locale` / `LANG`…), else English |
| `en` | English |
| `zh_cn` | Simplified Chinese |

Missing messages: **active → en → message id**.

## Conventions

- Hyphenated ids (`settings-language`, `filter-active`)
- Prefer `message_key()` + `shell.t(key)`; placeholders `{ $n }`, never concatenate fragments
- Schema **v8+** `language`; Settings → General saves and hot-swaps `Translator`

## Add a string

1. Same id in **both** `en/*.ftl` and `zh-CN/*.ftl`
2. Call from UI via `t("…")`
3. `cargo test -p ariadeck-i18n` (key parity)

## Catalogs

| File | Content |
| --- | --- |
| `app.ftl` | Filters, status, connection, chrome |
| `settings.ftl` | Settings |
| `chrome.ftl` | Empty states, toolbar |
| `dialogs.ftl` | Add/remove/details |
| `notices.ftl` | Toasts / command feedback |
| `errors.ftl` | Error codes |

## Errors

Stable codes (`ApplicationErrorCode::as_str`, e.g. `rpc.timeout`) → Fluent `error-` + dots→hyphens (`error-rpc-timeout`) via `OperationErrorView::localized_summary`. Missing key → English application summary.

## Out of scope (now)

Full migration of every validation string · Traditional Chinese / other locales · Application depending on Fluent (map codes in UI)

See residual i18n work in [`roadmap.md`](roadmap.md) Phase A2.
