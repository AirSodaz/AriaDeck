# AriaDeck i18n

## Stack

- **Fluent** (`.ftl`) via `fluent-bundle`
- Crate: `ariadeck-i18n` (no GPUI)
- Embedded catalogs under `crates/ariadeck-i18n/i18n/{en,zh-CN}/*.ftl`

## Locales

| Preference (settings) | Resolved catalog |
| --- | --- |
| `system` | OS UI language via `sys-locale` (Windows preferred UI languages), then POSIX `LANG`/`LC_*`; else English |
| `en` | English |
| `zh_cn` | Simplified Chinese |

Fallback chain for missing messages: **active locale â†’ en â†’ message id**.

## Message ids

- Hyphenated Fluent ids (e.g. `settings-language`, `filter-active`)
- Prefer keys on view enums via `message_key()` + `shell.t(key)`
- Do not concatenate translated fragments for word order; use Fluent placeholders (`{ $n }`)

## Adding a string

1. Add the same id to **both** `en/*.ftl` and `zh-CN/*.ftl`
2. Call `self.t("my-key")` from UI (or pass through a helper)
3. Run `cargo test -p ariadeck-i18n` (catalog key parity)

## Settings

- Schema **v8** adds `language`
- Changing language in **Settings â†’ General** saves immediately and reloads the process `Translator`

## Out of scope (for now)

- Full migration of every notice/dialog/application validation string
- Traditional Chinese / other locales
- Application layer depending on Fluent (prefer stable error codes + UI mapping later)

## Error codes

Application errors carry stable codes via `ApplicationErrorCode::as_str()` (e.g. `rpc.timeout`).
UI maps them to Fluent ids with `error-` + dotsâ†’hyphens (`error-rpc-timeout`) through `OperationErrorView::localized_summary`.
If a key is missing, the English `summary` from the engine/application layer is shown.
